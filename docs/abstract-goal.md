Я занимаюсь разработкой проекта для активного отразила жизни - momentum. Данный проект представляет из себя совокупность элементов для контроля и анализа активного образа жизни, регистрации результатов и достижений, социального взаимодействия с участниками платформы для достижения общих целей и также представляет из себя платформу для личных тренеров и диетологов для составления и набор групп участников, и удобной организации рабочего процесса. 

Другими словами: есть цель создать социальную сеть для людей с активным образом жизни, которые хотят все поменять в лучшую сторону. В этом «Life Hub” будет реализована система Habit Tracking, для удобного учета прогресса. Помимо этого будет включена возможность создавать команды для совместного использования со своими друзьями и общих побед. Ещё эта площадка будет представлять из себя возможность другим участникам платформы найти компанию благодарю системе Feed с рекомендациями людей поблизости и мероприятий которые могут устраивать участники. Например если кому-то нужна компания для совместных пробежек, то всегда можно будет оставить отзыв с подходящими характеристиками темпа бега и найти попутчика! Но если пользователю не нужен весь социальный функционал, то всегда можно использовать приложение в личных целях, для отслеживания личного прогресса в спорте (будут созданы определенные сферы до логирования: плавание, бег, велоспорт, тренажерный зал…). В каждой сфере можно будет вести записи своего прогресса и видеть аналитику. Также у каждого участника будет свой профиль и с возможностью публикации постов со своими результатами или другими интересностями своей активной жизни. Также будет создана страница для подсчета веса или питания. Помимо этого, будет проведена работа над созданием ИИ ассистента и таблицы для контроля привычек. Все это и будет приложением - Momentum. 

В социальной части будут введены определенные роли: 

- участник (роль по умолчанию для всех)
- владелец команды (участник может быть владельцем команд когда их создает)
- администратор команды (участник команды может быть администратором команды в которой состоит, если такая роль была присвоена ему владельцем команды)

Помимо всего функционала выше, будет реализована система мероприятий и соревнований.

Мероприятия могут быть публичными, для команды, или для выбранных участников. Под мероприятиям можно понимать отзыв на совместную пробежку, как это было расписано выше. Или на совместный поход с друзьями. Для того чтобы выкинуть оповещение, достаточно только создать мероприятие и режим видимости для остальных участников. 

Соревнования это уже отдельный вид событий которое может длится неограниченное количество времени и в котором могут принимать участие участники или команды.

При чем можно будет настраивать количество необходимых участников в команде для участия или выбрать участие всех только по одиночке без команд. Соревнования могут проходить по выбранным сферам (бег, плавание…) и записями результатов по команде или личных, если участвует только один участник.  

Однако необходимо будет продумать систему контроля честности, но ее в любом случае не получится довести до идеала никогда, поэтому участие будет основано на доверии.
