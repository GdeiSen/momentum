# 5 Руководство пользователя

## 5.1 Руководство для конечного пользователя

### 5.1.1 Регистрация и авторизация

Для начала работы с приложением Momentum необходимо создать учетную запись. Процесс регистрации является простым и интуитивно понятным, требуя минимального количества информации от пользователя.

Для регистрации нового аккаунта необходимо открыть главную страницу приложения в веб-браузере. На главной странице отображается информация о возможностях приложения и кнопка для перехода к регистрации. Нажатие на кнопку "Зарегистрироваться" или "Get Started" перенаправляет на страницу регистрации.

На странице регистрации отображается форма с тремя полями. Первое поле предназначено для ввода адреса электронной почты. Email будет использоваться для входа в систему и восстановления доступа к аккаунту при необходимости. Необходимо ввести действующий email адрес, к которому имеется доступ. Второе поле предназначено для создания пароля. Пароль должен содержать минимум восемь символов. Рекомендуется использовать комбинацию букв, цифр и специальных символов для обеспечения безопасности аккаунта. Третье поле предназначено для выбора уникального никнейма. Никнейм будет отображаться другим пользователям и использоваться в URL профиля. Никнейм должен содержать от трех до тридцати символов и может включать буквы, цифры, дефисы и подчеркивания.

После заполнения всех полей необходимо нажать кнопку "Зарегистрироваться". Система выполнит валидацию введенных данных. Если все данные корректны и email с никнеймом не заняты другими пользователями, аккаунт будет создан. Пользователь автоматически войдет в систему и будет перенаправлен на страницу профиля. Если возникли ошибки, под соответствующими полями отобразятся сообщения с описанием проблемы. Необходимо исправить ошибки и повторить попытку регистрации.

Для входа в существующий аккаунт необходимо перейти на страницу входа, нажав кнопку "Войти" на главной странице или в шапке сайта. На странице входа отображается форма с двумя полями. Первое поле предназначено для ввода email адреса, указанного при регистрации. Второе поле предназначено для ввода пароля. После заполнения полей необходимо нажать кнопку "Войти". Система проверит учетные данные. При успешной проверке пользователь будет перенаправлен на страницу профиля. При неверных учетных данных отобразится сообщение об ошибке.

Система автоматически сохраняет сессию пользователя, что позволяет оставаться в системе при закрытии и повторном открытии браузера. Сессия действительна в течение семи дней. После истечения этого срока потребуется повторный вход. Для выхода из системы необходимо открыть меню пользователя, нажав на аватар в правом верхнем углу, и выбрать пункт "Выйти". Это завершит текущую сессию и перенаправит на главную страницу.

### 5.1.2 Работа с профилем

Страница профиля является главной страницей пользователя после входа в систему. На этой странице отображается персонализированная информация и предоставляется доступ к основным функциям приложения.

В верхней части страницы профиля отображается приветственное сообщение с никнеймом пользователя и мотивационный текст. Ниже расположены виджеты с ключевой статистикой. Виджет "Current Streak" показывает текущую серию последовательных дней выполнения привычек. Виджет "Completed Habits" отображает общее количество выполненных привычек за все время. Виджет "Active Teams" показывает количество команд, в которых пользователь является участником. Виджет "Challenges" отображает количество челленджей, в которых пользователь участвует.

Виджет "My Teams" отображает список команд пользователя с возможностью быстрого перехода на страницу команды. Для каждой команды показывается название, аватар и количество участников. Нажатие на карточку команды перенаправляет на страницу этой команды. Если команд больше пяти, отображается кнопка "View all teams" для перехода к полному списку.

Пользователь может настроить расположение виджетов на странице профиля. На десктопных устройствах доступна функция перетаскивания виджетов. Для перемещения виджета необходимо навести на него курсор, зажать левую кнопку мыши и перетащить виджет в желаемое место. Виджеты можно размещать в трех областях: фиксированные слоты в верхней части страницы, свободная рабочая область в центре и док-панель внизу страницы. Все изменения автоматически сохраняются на сервере и применяются на всех устройствах пользователя.

Для редактирования профиля необходимо открыть меню пользователя, нажав на аватар в правом верхнем углу, и выбрать пункт "Настройки" или "Profile Settings". На странице настроек отображаются поля для редактирования информации профиля. Поле "Nickname" позволяет изменить отображаемое имя. Изменение никнейма доступно не чаще одного раза в тридцать дней. Поле "Bio" позволяет добавить или изменить краткую биографию, которая будет отображаться в профиле. Биография может содержать до пятисот символов.

Для загрузки аватара необходимо нажать на текущий аватар или кнопку "Upload Avatar". Откроется диалог выбора файла. Необходимо выбрать изображение в формате JPEG, PNG, GIF или WebP размером до пяти мегабайт. После выбора файла начнется загрузка. По завершении загрузки новый аватар отобразится в профиле. Аналогично можно загрузить фоновое изображение профиля через кнопку "Upload Header Background".

Для изменения пароля необходимо перейти в раздел "Security" в настройках. Отображается форма с тремя полями. Первое поле "Current Password" требует ввода текущего пароля для подтверждения личности. Второе поле "New Password" предназначено для ввода нового пароля. Третье поле "Confirm New Password" требует повторного ввода нового пароля для предотвращения ошибок. После заполнения всех полей необходимо нажать кнопку "Change Password". Система проверит текущий пароль и обновит его на новый при успешной проверке.

В разделе "Sessions" отображается список всех активных сессий пользователя. Для каждой сессии показывается информация о браузере, операционной системе, IP-адресе и дате создания. Пользователь может отозвать любую сессию, нажав кнопку "Revoke" рядом с ней. Это полезно, если пользователь забыл выйти из системы на чужом устройстве или подозревает несанкционированный доступ. Кнопка "Logout from all devices" завершает все сессии, кроме текущей, что обеспечивает принудительный выход со всех устройств.

### 5.1.3 Работа с командами

Команды являются основной единицей совместной работы в приложении Momentum. Пользователи могут создавать команды, присоединяться к существующим командам и участвовать в совместных челленджах.

Для просмотра списка команд необходимо перейти в раздел "Teams" через навигационное меню. Отображается страница с двумя вкладками. Вкладка "My Teams" показывает команды, в которых пользователь является участником. Для каждой команды отображается карточка с названием, слоганом, описанием, количеством участников и количеством челленджей. Нажатие на карточку перенаправляет на страницу команды. Вкладка "Find Teams" показывает все публичные команды, к которым пользователь может присоединиться. В верхней части вкладки расположено поле поиска для фильтрации команд по названию или описанию.

Для создания новой команды необходимо нажать кнопку "Create Team" в верхней части страницы команд. Откроется форма создания команды. Поле "Name" является обязательным и должно содержать название команды от трех до пятидесяти символов. Поле "Slogan" является опциональным и может содержать краткий слоган команды до ста символов. Поле "Description" является опциональным и может содержать подробное описание команды до пятисот символов.

Чекбокс "Private Team" определяет, будет ли команда отображаться в общем списке команд. Если чекбокс установлен, команда будет приватной и не будет видна другим пользователям. Присоединение к приватной команде возможно только по коду приглашения. Чекбокс "Require Invite Code" определяет, требуется ли код приглашения для присоединения к команде. Если чекбокс установлен, пользователи смогут присоединиться только по валидному коду. Чекбокс "Require Email Whitelist" определяет, требуется ли наличие email пользователя в whitelist для присоединения.

После заполнения формы необходимо нажать кнопку "Create Team". Команда будет создана, и пользователь будет автоматически добавлен как владелец команды. Произойдет перенаправление на страницу созданной команды.

Для присоединения к публичной команде необходимо найти команду на вкладке "Find Teams". Если команда не требует код приглашения и не требует whitelist email, рядом с карточкой команды отображается кнопка "Join". Нажатие на кнопку добавляет пользователя в команду с ролью обычного участника. Произойдет перенаправление на страницу команды.

Для присоединения по коду приглашения необходимо нажать кнопку "Enter Code" в верхней части страницы команд. Откроется диалог для ввода шестисимвольного кода приглашения. Необходимо ввести код, полученный от администратора или владельца команды. После ввода всех шести символов система автоматически валидирует код. Если код валиден, отображается информация о команде с кнопкой "Join Team". Нажатие на кнопку добавляет пользователя в команду.

Для выхода из команды необходимо открыть страницу команды и нажать кнопку с иконкой настроек в правом верхнем углу. В выпадающем меню выбрать пункт "Leave Team". Отобразится диалог подтверждения с предупреждением о том, что выход из команды прекратит доступ ко всем челленджам и контенту команды. Необходимо подтвердить действие, нажав кнопку "Leave". Пользователь будет удален из команды и перенаправлен на страницу списка команд. Владелец команды не может покинуть команду, он должен либо передать владение другому участнику, либо удалить команду.

### 5.1.4 Работа с челленджами

Челленджи представляют собой совместные испытания с определенным периодом проведения и набором привычек для отслеживания. Участие в челленджах мотивирует к регулярному выполнению привычек через социальную поддержку и элементы соревнования.

Для просмотра челленджей команды необходимо открыть страницу команды. В верхней части страницы отображается горизонтальная лента с постерами челленджей. Каждый постер представляет один челлендж и показывает его название и статус. Статус может быть "Active" для текущих челленджей, "Upcoming" для будущих и "Completed" для завершенных. Нажатие на постер перенаправляет на страницу челленджа.

Альтернативно можно просмотреть все челленджи в галерее, перейдя в раздел "Challenges" через навигационное меню. Отображается страница с 3D визуализацией постеров в верхней части и сеткой челленджей ниже. В верхней части страницы находится поле поиска для фильтрации челленджей по названию, описанию или названию команды. Каждая карточка челленджа показывает постер, название, описание, статус и название команды. Нажатие на карточку перенаправляет на страницу челленджа.

На странице челленджа отображается детальная информация. В заголовке показывается название челленджа, описание, статус, даты начала и окончания, количество привычек и количество участников. Для администраторов и владельцев команды доступны кнопки для редактирования и удаления челленджа, а также для добавления новых привычек.

Основным элементом страницы является таблица отслеживания привычек. Таблица представляет собой сетку, где строки соответствуют привычкам, а столбцы - датам в периоде челленджа. Каждая ячейка представляет выполнение конкретной привычки в конкретную дату. Ячейки окрашены в соответствии со статусом выполнения. Зеленый цвет означает полное выполнение. Желтый цвет означает частичное выполнение. Серый цвет означает пропуск с разрешением администратора. Пустая ячейка означает отсутствие записи выполнения.

Для отметки выполнения привычки необходимо найти соответствующую ячейку в таблице и нажать на нее. Откроется контекстное меню с опциями. Опция "Completed" отмечает привычку как полностью выполненную. Опция "Partial" отмечает привычку как частично выполненную. Опция "Clear" удаляет отметку выполнения. Для администраторов и владельцев доступна дополнительная опция "Skipped Excused" для отметки пропуска с разрешением для любого участника. Выбор опции обновляет статус ячейки, и изменение сохраняется на сервере.

В верхней части таблицы находится выпадающий список для фильтрации по пользователю. По умолчанию отображаются логи всех участников. Выбор конкретного пользователя отфильтрует таблицу, показывая только его логи. Это полезно для просмотра прогресса конкретного участника.

Ниже таблицы отображается 3D визуализация лидерборда в виде горы. Участники представлены как точки на поверхности горы, где высота соответствует их проценту выполнения. Топ-пять участников отображаются с рангами. Гора автоматически вращается для демонстрации всех участников. Пользователь может отключить автоматическое вращение и управлять камерой вручную с помощью мыши.

Под 3D визуализацией отображается таблица лидерборда с детальной информацией. Для каждого участника показывается позиция, аватар, никнейм, количество выполненных привычек, общее количество логов и процент выполнения. Текущий пользователь выделяется меткой "вы" для быстрой идентификации своей позиции.

В правой части страницы отображаются статистические карточки. Карточка "Progress" показывает количество прошедших дней от общего количества дней челленджа с прогресс-баром. Карточка "Completion" показывает процент выполнения с количеством выполненных привычек. Карточка "Partial" показывает количество частично выполненных привычек. Карточка "Excused" показывает количество пропущенных с разрешением привычек.

### 5.1.5 Отслеживание привычек

Регулярное отслеживание привычек является ключевым аспектом использования приложения Momentum. Система предоставляет несколько способов отметки выполнения привычек для удобства пользователей.

Основной способ отслеживания привычек - через таблицу на странице челленджа, как описано в предыдущем разделе. Этот способ подходит для детального просмотра и редактирования логов за различные даты. Альтернативный способ - через виджет "Today's Habits" на странице профиля. Виджет отображает список всех привычек, которые необходимо выполнить сегодня, из всех активных челленджей пользователя.

Для каждой привычки в виджете показывается название привычки, название челленджа и название команды. Рядом с каждой привычкой находятся кнопки для быстрой отметки статуса. Кнопка с иконкой галочки отмечает привычку как выполненную. Кнопка с иконкой половинной галочки отмечает привычку как частично выполненную. Нажатие на кнопку обновляет статус и отправляет изменение на сервер. Кнопка меняет цвет в соответствии с установленным статусом для визуальной обратной связи.

Для просмотра истории выполнения привычек необходимо перейти в раздел "History" в настройках или профиле. Отображается календарное представление с цветовой кодировкой дней. Дни с выполненными привычками окрашены в зеленый цвет с интенсивностью, соответствующей количеству выполненных привычек. Дни без выполненных привычек остаются без цвета. Нажатие на день показывает детальную информацию о выполненных привычках в этот день.

Под календарем отображается график с динамикой выполнения привычек за выбранный период. График показывает количество выполненных, частично выполненных и общее количество привычек по дням. Это позволяет визуально оценить прогресс и выявить периоды снижения активности.

Для добавления заметки к отметке выполнения необходимо открыть детальный вид лога. На странице челленджа нажать на ячейку таблицы правой кнопкой мыши или долгим нажатием на мобильных устройствах. Откроется расширенное меню с опцией "Add Note". Выбор опции откроет диалог с текстовым полем. Необходимо ввести заметку до пятисот символов и нажать кнопку "Save". Заметка сохранится вместе с логом и будет доступна при наведении на ячейку или в детальном просмотре.

Система автоматически рассчитывает текущую серию последовательных дней выполнения привычек. Серия отображается в виджете "Current Streak" на странице профиля. Серия увеличивается на единицу за каждый день, когда выполнена хотя бы одна привычка. Серия прерывается, если в течение дня не было выполнено ни одной привычки. Дни с пропуском с разрешением администратора не прерывают серию. Максимальная достигнутая серия также отслеживается и отображается в статистике пользователя.

### 5.1.6 Просмотр статистики

Приложение Momentum предоставляет детальную статистику для анализа прогресса и мотивации к продолжению работы над привычками. Статистика доступна на нескольких уровнях: личная статистика пользователя, статистика команды и статистика челленджа.

Личная статистика пользователя отображается на странице профиля в виде виджетов. Виджет "Current Streak" показывает текущую серию последовательных дней с выполнением привычек. Виджет "Completed Habits" показывает общее количество выполненных привычек за все время использования приложения. Виджет "Active Teams" показывает количество команд, в которых пользователь является участником. Виджет "Challenges" показывает количество челленджей, в которых пользователь участвует или участвовал.

Для просмотра детальной статистики необходимо перейти в раздел "Statistics" через меню пользователя. Отображается страница с несколькими разделами. Раздел "Overview" показывает общую статистику с графиками активности за различные периоды. График "Daily Activity" показывает количество выполненных привычек по дням за последний месяц. График "Weekly Summary" показывает суммарную статистику по неделям за последние три месяца. График "Monthly Trends" показывает тренды выполнения за последний год.

Раздел "Habits Breakdown" показывает разбивку по типам привычек или категориям, если они определены. Для каждой категории отображается процент выполнения, количество выполненных привычек и средняя частота выполнения. Раздел "Achievements" показывает достигнутые вехи, такие как первая выполненная привычка, первая серия из семи дней, сто выполненных привычек и другие. Для каждого достижения показывается иконка, название, описание и дата получения.

Статистика команды отображается на странице команды. В верхней части страницы показываются ключевые метрики команды. Количество участников показывает текущее количество членов команды. Количество челленджей показывает общее количество созданных челленджей. Количество активных челленджей показывает челленджи, проходящие в данный момент. Общее количество выполненных привычек показывает суммарный результат всех участников команды.

График активности команды показывает динамику выполнения привычек всеми участниками за выбранный период. Можно переключаться между периодами: неделя, месяц, год. График обновляется в соответствии с выбранным периодом. Радарный график показывает активность по недавним челленджам. Для каждого челленджа отображается процент выполнения в виде многоугольника. Это позволяет визуально сравнить активность в различных челленджах.

Статистика челленджа отображается на странице челленджа в виде карточек и графиков. Карточка "Progress" показывает прогресс челленджа с количеством прошедших дней и прогресс-баром. Карточка "Completion" показывает общий процент выполнения челленджа всеми участниками. Карточка "Participants" показывает количество активных участников челленджа. Карточка "Habits" показывает количество привычек в челлендже.

Лидерборд челленджа показывает рейтинг участников по результатам выполнения. Для каждого участника показывается позиция, аватар, никнейм, количество выполненных привычек, общий счет и процент выполнения. Счет вычисляется как сумма полностью выполненных привычек и половины частично выполненных. Участники сортируются по убыванию счета. При равенстве счетов применяется дополнительная сортировка по текущей серии.

Для экспорта статистики в файл необходимо перейти в раздел "Export" в настройках статистики. Доступны несколько форматов экспорта. Формат CSV подходит для импорта в электронные таблицы. Формат JSON подходит для программной обработки данных. Формат PDF подходит для печати и архивирования. Необходимо выбрать формат, период данных для экспорта и нажать кнопку "Export". Файл будет сгенерирован и автоматически загружен в браузер.

## 5.2 Руководство для администратора команды

### 5.2.1 Управление командой

Администраторы и владельцы команд имеют расширенные права для управления командой и ее контентом. Владелец команды имеет полный контроль над всеми аспектами команды, включая возможность удаления команды и изменения параметров приватности. Администраторы имеют права на управление контентом, но не могут изменять критические настройки команды.

Для редактирования настроек команды необходимо открыть страницу команды и нажать кнопку "Edit Team" в правом верхнем углу. Откроется форма редактирования с полями для изменения информации о команде. Поле "Name" позволяет изменить название команды. Поле "Slogan" позволяет изменить или добавить слоган. Поле "Description" позволяет изменить или добавить описание команды.

Для загрузки фонового изображения команды необходимо нажать кнопку "Upload Header Background". Откроется диалог выбора файла. Необходимо выбрать изображение в поддерживаемом формате размером до пяти мегабайт. После выбора файла начнется загрузка. По завершении загрузки новое фоновое изображение отобразится на странице команды.

Чекбоксы для параметров приватности доступны только владельцу команды. Чекбокс "Private Team" определяет видимость команды в общем списке. Чекбокс "Require Invite Code" определяет необходимость кода приглашения для присоединения. Чекбокс "Require Email Whitelist" определяет необходимость наличия email в whitelist. Изменение этих параметров влияет на способы присоединения к команде новых участников.

После внесения изменений необходимо нажать кнопку "Save Changes". Изменения будут сохранены на сервере и отобразятся на странице команды. Для отмены изменений необходимо нажать кнопку "Cancel" или закрыть форму без сохранения.

Для удаления команды необходимо открыть меню настроек команды и выбрать пункт "Delete Team". Эта опция доступна только владельцу команды. Отобразится диалог подтверждения с предупреждением о необратимости операции. Удаление команды приведет к удалению всех связанных данных, включая челленджи, привычки, логи выполнения, посты и сообщения. Необходимо ввести название команды для подтверждения и нажать кнопку "Delete". Команда будет удалена, и пользователь будет перенаправлен на страницу списка команд.

Для передачи владения командой другому участнику необходимо открыть список участников команды. Найти участника, которому необходимо передать владение. Нажать на иконку настроек рядом с участником и выбрать пункт "Transfer Ownership". Отобразится диалог подтверждения с предупреждением о том, что после передачи владения текущий владелец станет администратором. Необходимо подтвердить действие, нажав кнопку "Transfer". Владение будет передано выбранному участнику, и его роль изменится на OWNER, а роль текущего владельца изменится на ADMIN.

### 5.2.2 Управление челленджами

Администраторы и владельцы команд могут создавать, редактировать и удалять челленджи для организации совместных испытаний.

Для создания нового челленджа необходимо открыть страницу команды и нажать кнопку "Create Challenge". Откроется форма создания челленджа. Поле "Title" является обязательным и должно содержать название челленджа от трех до ста символов. Поле "Description" является опциональным и может содержать подробное описание целей и правил челленджа.

Поле "Start Date" определяет дату начала челленджа. Необходимо выбрать дату из календаря или ввести вручную в формате YYYY-MM-DD. Поле "End Date" определяет дату окончания челленджа. Дата окончания должна быть позже даты начала. Система автоматически валидирует это условие.

Для загрузки постера челленджа необходимо нажать кнопку "Upload Poster". Откроется диалог выбора файла. Постер используется для визуального представления челленджа на странице команды и в галерее челленджей. Рекомендуется использовать изображение с соотношением сторон шестнадцать к девяти для оптимального отображения.

После заполнения формы необходимо нажать кнопку "Create Challenge". Челлендж будет создан, и произойдет перенаправление на страницу челленджа. На странице челленджа можно добавить привычки, нажав кнопку "Add Habit". Откроется форма с полями "Title" и "Description" для привычки. После заполнения формы необходимо нажать кнопку "Add". Привычка будет добавлена в челлендж и отобразится в таблице отслеживания.

Для редактирования челленджа необходимо открыть страницу челленджа и нажать кнопку "Edit Challenge". Откроется форма редактирования с текущими значениями полей. Можно изменить название, описание, даты и постер. После внесения изменений необходимо нажать кнопку "Save Changes". Изменения будут сохранены и отобразятся на странице челленджа.

Для удаления челленджа необходимо открыть меню настроек челленджа и выбрать пункт "Delete Challenge". Отобразится диалог подтверждения с предупреждением о том, что удаление челленджа приведет к удалению всех привычек и логов выполнения. Необходимо подтвердить действие, нажав кнопку "Delete". Челлендж будет удален, и пользователь будет перенаправлен на страницу команды.

Для редактирования привычки необходимо нажать на название привычки в таблице отслеживания. Откроется форма редактирования с полями "Title" и "Description". После внесения изменений необходимо нажать кнопку "Save". Для удаления привычки необходимо нажать кнопку "Delete" в форме редактирования. Отобразится диалог подтверждения. Необходимо подтвердить удаление. Привычка и все ее логи будут удалены.

### 5.2.3 Управление участниками

Администраторы и владельцы команд могут управлять участниками, изменять их роли и удалять участников из команды.

Для просмотра списка участников необходимо открыть страницу команды и прокрутить до раздела "Members". Отображается таблица со всеми участниками команды. Для каждого участника показывается аватар, никнейм и роль. Роли обозначаются иконками: корона для владельца, щит для администратора, отсутствие иконки для обычного участника.

Для изменения роли участника необходимо нажать на иконку настроек рядом с участником. Откроется меню с опциями. Опция "Make Admin" доступна владельцу для назначения участника администратором. Опция "Remove Admin" доступна владельцу для снятия роли администратора. Опция "Remove Member" доступна администраторам и владельцу для удаления участника из команды.

Выбор опции "Make Admin" изменяет роль участника на ADMIN. Администратор получает права на создание и редактирование челленджей, управление привычками и модерацию контента. Выбор опции "Remove Admin" изменяет роль администратора обратно на MEMBER. Участник теряет расширенные права и становится обычным членом команды.

Выбор опции "Remove Member" удаляет участника из команды. Отобразится диалог подтверждения. Необходимо подтвердить удаление, нажав кнопку "Remove". Участник будет удален из team_members, что прекратит его доступ ко всем ресурсам команды. Логи выполнения привычек удаленного участника сохраняются для целостности статистики челленджей.

Ограничения на управление участниками включают следующие правила. Владельца команды нельзя удалить из команды. Администратор не может изменить роль или удалить другого администратора, это может сделать только владелец. Администратор не может изменить роль или удалить владельца. Участник не может изменять роли или удалять других участников.

Для просмотра статистики участника необходимо нажать на его аватар или никнейм в списке участников. Откроется страница профиля участника с его статистикой в контексте команды. Отображается количество выполненных привычек в челленджах команды, текущая серия, процент выполнения и позиция в лидербордах различных челленджей.

### 5.2.4 Система приглашений

Система приглашений позволяет администраторам и владельцам команд контролировать присоединение новых участников через коды приглашений и whitelist email.

Для создания кода приглашения необходимо открыть настройки команды и перейти в раздел "Invites". Нажать кнопку "Create Invite Code". Откроется форма с опциональными параметрами. Поле "Max Uses" определяет максимальное количество использований кода. Если оставить поле пустым, код можно использовать неограниченное количество раз. Поле "Expires At" определяет дату истечения кода. Если оставить поле пустым, код будет действителен бессрочно.

После заполнения формы необходимо нажать кнопку "Create". Система сгенерирует уникальный шестисимвольный код и отобразит его в списке кодов приглашений. Код можно скопировать, нажав кнопку "Copy" рядом с ним. Скопированный код можно отправить потенциальным участникам через любые каналы связи.

Для каждого кода в списке отображается сам код, дата создания, количество использований, максимальное количество использований, дата истечения и статус активности. Кнопка "Deactivate" позволяет деактивировать код без его удаления. Деактивированный код нельзя использовать для присоединения, но он остается в списке для истории. Кнопка "Delete" полностью удаляет код из системы.

Для управления whitelist email необходимо перейти в раздел "Whitelist" в настройках команды. Отображается список всех email адресов, которым разрешено присоединение к команде. Для добавления одного email необходимо ввести адрес в поле "Email" и нажать кнопку "Add". Email будет добавлен в whitelist.

Для массового добавления email необходимо нажать кнопку "Bulk Add". Откроется диалог с текстовым полем для ввода нескольких email адресов. Необходимо ввести email адреса, разделяя их запятыми или новыми строками. После ввода необходимо нажать кнопку "Add All". Система обработает список и добавит все валидные email в whitelist. Дублирующиеся email будут пропущены. По завершении отобразится статистика: количество добавленных, пропущенных и общее количество.

Для удаления email из whitelist необходимо нажать кнопку "Remove" рядом с email в списке. Отобразится диалог подтверждения. Необходимо подтвердить удаление. Email будет удален из whitelist, и пользователи с этим email больше не смогут присоединиться к команде, если требуется whitelist.

Комбинация параметров приватности, требования кода приглашения и требования whitelist email позволяет гибко настроить доступ к команде. Публичная команда без требований доступна всем для присоединения. Публичная команда с требованием кода доступна всем, кто знает код. Публичная команда с требованием whitelist доступна только пользователям с разрешенными email. Приватная команда с требованием кода доступна только по коду и не отображается в списке. Приватная команда с требованием whitelist и кода требует и код, и наличие email в whitelist.

## 5.3 Руководство по развёртыванию приложения

### 5.3.1 Требования к системе

Для развертывания приложения Momentum необходима система, соответствующая минимальным требованиям по аппаратному и программному обеспечению.

Минимальные аппаратные требования включают процессор с двумя ядрами и частотой минимум два гигагерца. Оперативная память минимум четыре гигабайта для комфортной работы всех компонентов. Дисковое пространство минимум двадцать гигабайт для установки всех компонентов и хранения данных. Сетевое подключение с пропускной способностью минимум десять мегабит в секунду.

Рекомендуемые аппаратные требования для продакшен окружения включают процессор с четырьмя ядрами и частотой минимум три гигагерца. Оперативная память восемь гигабайт или более. Дисковое пространство пятьдесят гигабайт или более с использованием SSD для улучшения производительности базы данных. Сетевое подключение с пропускной способностью сто мегабит в секунду или более.

Программные требования включают операционную систему Linux, предпочтительно Ubuntu версии двадцать или выше, либо другой дистрибутив с поддержкой Docker. Альтернативно можно использовать macOS или Windows с установленным Docker Desktop. Docker версии двадцать или выше для контейнеризации приложения. Docker Compose версии два или выше для оркестрации контейнеров. Git для клонирования репозитория с исходным кодом.

Для разработки дополнительно требуется Node.js версии двадцать или выше для запуска приложения без Docker. npm или yarn для управления зависимостями. Текстовый редактор или IDE, такой как Visual Studio Code, для редактирования кода.

### 5.3.2 Подготовка к развёртыванию

Процесс подготовки к развертыванию включает несколько этапов настройки окружения и конфигурации приложения.

Первым этапом является клонирование репозитория с исходным кодом. Необходимо открыть терминал и выполнить команду git clone с URL репозитория. После завершения клонирования необходимо перейти в директорию проекта с помощью команды cd.

Вторым этапом является создание файлов конфигурации. В корневой директории проекта необходимо создать файл .env на основе файла .env.example. Файл .env содержит переменные окружения для настройки приложения. Необходимо открыть файл в текстовом редакторе и заполнить следующие переменные.

Переменная DATABASE_URL определяет строку подключения к PostgreSQL в формате postgresql://username:password@host:port/database. Для Docker окружения host должен быть postgres, что соответствует имени сервиса в docker-compose.yml. Переменные JWT_ACCESS_SECRET и JWT_REFRESH_SECRET определяют секретные ключи для подписи токенов. Необходимо сгенерировать случайные строки длиной минимум тридцать два символа. Можно использовать онлайн генератор или команду openssl rand -base64 32 в терминале.

Переменные JWT_ACCESS_EXPIRATION и JWT_REFRESH_EXPIRATION определяют время жизни токенов. Рекомендуемые значения: пятнадцать минут для access token и семь дней для refresh token. Значения указываются в формате, понятном библиотеке ms, например 15m или 7d.

Переменные для MinIO включают MINIO_ENDPOINT, MINIO_PORT, MINIO_USE_SSL, MINIO_ACCESS_KEY, MINIO_SECRET_KEY и MINIO_BUCKET. Для Docker окружения endpoint должен быть minio. Порт по умолчанию девять тысяч. USE_SSL должен быть false для локального развертывания. Access key и secret key необходимо установить в соответствии с конфигурацией MinIO в docker-compose.yml. Bucket name можно оставить по умолчанию momentum-files.

Переменная PORT определяет порт, на котором будет запущен серверный API. По умолчанию три тысячи один. Переменная NEXT_PUBLIC_API_URL определяет URL серверного API для клиентского приложения. Для локального развертывания значение http://localhost:3001.

Третьим этапом является настройка Docker Compose. Файл docker-compose.yml уже содержит конфигурацию всех сервисов. При необходимости можно изменить порты, пробрасываемые на хост, или параметры контейнеров. Для продакшен окружения рекомендуется изменить пароли баз данных и ключи доступа MinIO на более безопасные значения.

### 5.3.3 Запуск приложения

После завершения подготовки можно приступить к запуску приложения.

Для запуска с использованием Docker Compose необходимо открыть терминал в корневой директории проекта. Выполнить команду docker-compose up для запуска всех сервисов. При первом запуске Docker загрузит необходимые образы и соберет образы для серверного и клиентского приложений. Это может занять несколько минут в зависимости от скорости интернет-соединения и производительности системы.

После завершения сборки и запуска контейнеров в терминале отобразятся логи всех сервисов. Необходимо дождаться сообщений о успешном запуске. Для PostgreSQL это сообщение "database system is ready to accept connections". Для MinIO это сообщение "API: http://minio:9000". Для серверного приложения это сообщение "Nest application successfully started". Для клиентского приложения это сообщение "ready - started server on".

После успешного запуска приложение доступно по следующим URL. Клиентское приложение доступно по адресу http://localhost:3000. Серверное API доступно по адресу http://localhost:3001. Документация API Swagger доступна по адресу http://localhost:3001/api/docs. Консоль управления MinIO доступна по адресу http://localhost:9001.

Для остановки приложения необходимо нажать Ctrl+C в терминале, где запущен Docker Compose. Это остановит все контейнеры. Для полного удаления контейнеров необходимо выполнить команду docker-compose down. Для удаления контейнеров вместе с volumes, что приведет к удалению всех данных, необходимо выполнить команду docker-compose down -v.

Для запуска в фоновом режиме необходимо выполнить команду docker-compose up -d. Флаг -d запускает контейнеры в detached режиме. Для просмотра логов необходимо выполнить команду docker-compose logs. Для просмотра логов конкретного сервиса необходимо указать его имя, например docker-compose logs server. Флаг -f позволяет следить за логами в реальном времени.

Для применения миграций базы данных необходимо выполнить команду внутри контейнера сервера. Команда docker-compose exec server npx prisma migrate deploy применяет все миграции к базе данных. Эту команду необходимо выполнить при первом запуске или после обновления схемы базы данных.

Для заполнения базы данных тестовыми данными можно выполнить команду docker-compose exec server npx prisma db seed. Это создаст несколько тестовых пользователей, команд и челленджей для демонстрации функциональности приложения.

### 5.3.4 Настройка и конфигурация

После успешного запуска приложения может потребоваться дополнительная настройка для оптимальной работы.

Для настройки CORS необходимо отредактировать файл momentum-server/src/main.ts. В методе enableCors можно указать конкретные домены, с которых разрешены запросы, вместо разрешения всех источников. Это повышает безопасность в продакшен окружении.

Для настройки SSL необходимо получить SSL сертификат для домена. Можно использовать бесплатный сертификат от Let's Encrypt. Необходимо настроить reverse proxy, такой как Nginx, для терминации SSL и проксирования запросов к приложению. Конфигурация Nginx должна включать настройки для проксирования запросов к клиентскому приложению на порту три тысячи и к серверному API на порту три тысячи один.

Для настройки резервного копирования необходимо настроить регулярное создание бэкапов базы данных PostgreSQL. Можно использовать утилиту pg_dump для создания дампов базы данных. Рекомендуется автоматизировать процесс через cron job, выполняющий бэкап ежедневно. Бэкапы необходимо хранить в безопасном месте, отдельно от сервера приложения.

Для настройки мониторинга можно использовать инструменты, такие как Prometheus и Grafana, для отслеживания метрик производительности. Необходимо настроить экспорт метрик из приложения и базы данных. Настроить дашборды в Grafana для визуализации метрик. Настроить алерты для уведомления о проблемах, таких как высокая нагрузка или ошибки.

Для масштабирования приложения можно увеличить количество реплик серверного приложения через Docker Compose или Kubernetes. Необходимо настроить load balancer для распределения нагрузки между репликами. Для базы данных можно настроить репликацию master-slave для распределения нагрузки чтения. Для файлового хранилища MinIO поддерживает распределенный режим для горизонтального масштабирования.
